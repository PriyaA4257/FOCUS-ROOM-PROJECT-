You are a senior full-stack developer tasked with building a complete, production-ready web application called “Focus Room – Virtual Study Space.” This application is designed to help students stay focused by studying together in real-time within virtual rooms, similar to a digital library environment. The system should allow users to register and securely log in using JWT-based authentication, after which they can create or join public and private study rooms.

Inside each room, users should experience a synchronized Pomodoro timer that runs consistently for all participants, enabling structured study sessions (25 minutes focus and 5 minutes break). The application should use WebSockets (STOMP over SockJS) to ensure real-time communication, including live updates when users join or leave rooms, synchronized timers, and optional room-based chat functionality. During focus sessions, a distraction-free “focus mode” should activate, minimizing the interface, disabling chat, and optionally detecting tab switching to encourage discipline.

The frontend should be built using React (Vite) with a clean, modern, mobile-first design that supports dark mode and smooth transitions. The UI should feel calming and minimal, with the timer prominently centered, a sidebar showing active users, and an optional collapsible chat panel. Axios should be used for API communication, and JWT tokens should be securely stored and managed.

The backend should be implemented using Java Spring Boot with a proper layered architecture including controllers, services, repositories, and models. PostgreSQL should be used for persistent storage, managing users, rooms, room participation, and study session history. The system should track user sessions, including start time, end time, and duration, and display this data in a personalized dashboard.

The dashboard should provide meaningful insights such as daily and weekly study time, number of completed sessions, consistency streaks, and most productive time periods. To further enhance the application, include features like a leaderboard to show top focused users, customizable study goals, sound notifications at the end of sessions, and a visually appealing progress graph. Users should also be able to personalize their experience with themes, avatars, and room backgrounds.

To make the application more engaging and polished, include subtle animations, smooth transitions, and responsive design across devices. Add small but impactful enhancements such as a “study streak” counter, motivational messages during breaks, and a quick room rejoin feature. The final product should feel like a real-world productivity tool rather than a basic student project.

Ensure the codebase is complete, well-structured, and follows best practices. Provide full frontend and backend implementation, database schema, WebSocket configuration, authentication setup, and clear instructions to run the application locally. The final result should be visually appealing, technically strong, and suitable for a final-year IT project demonstration.
